## 1. 書籍の概要

- 想定読者: インフラエンジニア（経験 1〜5年）を中心に、セキュリティ領域の知識を体系化したい読者（`docs/index.md`, `manuscript/chapter-introduction/index.md` より）。
- 本書のゴール: セキュリティ理論の理解に留まらず、インフラエンジニアが日常業務で「どう実装判断するか」を説明できる状態にする（概念→適用→実装→統合運用の段階設計）。
- 入口（読者導線）の現状: `README.md` と `docs/index.md` のライセンス表記は CC BY-NC-SA 4.0 で揃っている一方、章タイトルは `README.md` / `docs/index.md` と各章 front matter の `title:` の間で不一致があり、公開時の混乱が起きうる（要整理）。

## 3. 構成上の気になる点（候補）

- 入口（README / docs/index）と章タイトル表記の不整合:
  - `README.md` / `docs/index.md` の章タイトルと、各章 front matter の `title:` が一致していない箇所がある（読者導線の統一観点で課題）。
- 学習目標の見せ方が章ごとに不統一:
  - `manuscript/chapter-introduction/index.md` では「各章冒頭にある『この章で学ぶこと』」に言及しているが、本文側は章頭の学習目標を引用ブロック（`> -`）で提示しており、ラベル名が一致していない。
- 第8章と第9章のトピック境界が曖昧になりやすい:
  - 第8章にも「インシデント対応とフォレンジック」が含まれ、第9章でも同領域を扱うため、内容の重複や学習順の迷いが生じる可能性がある。
- 第7章の章タイトルと内容のズレ:
  - 章タイトルは「暗号化とデータ保護」だが、章内見出しはコンテナ/Kubernetes/DevSecOps中心であるため、タイトルから想起される内容と差がある（意図の確認が必要）。

## 4. 構成改善の提案（案）

- 入口の統一（公開を見据えた最優先）:
  - `README.md` を「読者入口」にするなら、`docs/index.md` と同一の書名・概要・章構成・ライセンスに合わせる（もしくは README は最小限にして `docs/index.md` へ誘導）。
  - ライセンス表記は CC BY-NC-SA 4.0 を正として README/オンライン版の双方に反映する（現状は揃っている）。
- 章タイトルの整合:
  - `docs/index.md`（目次）/ 各章 front matter `title:` / README の章タイトルを一致させ、引用・レビュー・検索の基準を揃える。
  - 第7章は特に「タイトルを本文に合わせる」または「本文の章内トピックをタイトルに合わせる」のどちらを採るかを決めてから編集する。
- Stage2要件（導入→本論→まとめ）の形式統一:
  - 章頭の学習目標を「この章で学ぶこと」ラベル付きに統一（`manuscript/chapter-introduction/index.md` の記述とも整合）。
  - 章末のまとめ見出しを「この章で学んだこと」等に統一するか、現状の「まとめ：…」形式を採用するかを決め、全章で一貫させる。
- 第8章/第9章の役割分担を明確化:
  - 第8章は「検知〜初動（SOC/SIEM/SOARと運用設計）」、第9章は「対応プロセスの体系化〜事後分析/成熟度」のように、章頭の学習目標と章末まとめで境界を明示する。
